# 01. Product Requirements Document (PRD)

## 1. 프로젝트 개요

### 1.1 기본 정보
| 항목 | 내용 |
|------|------|
| **프로젝트명** | 씨드림기프트 (W Gift) |
| **목표** | 백화점 상품권 판매 및 매입 플랫폼 |
| **참조 사이트** | yk24.shop (크롤링 데이터 기반 복제) |
| **플랫폼** | Windows Server 최적화 |

### 1.2 핵심 목표
1. **상품권 판매**: 사이트 → 고객 (결제 후 PIN 코드 자동 발송)

3. **KYC 인증**: 본인 확인 후 거래 허용
4. **권한 관리**: Admin, Partner, User 3단계 RBAC

## 2. 상세 요구사항 (30 Mandates)

### 2.1 인증 및 보안 (Mandates 1-5)

| # | 요구사항 | 설명 | 우선순위 |
|---|----------|------|----------|
| 1 | ID/PW 기반 회원가입 | 이메일 + 비밀번호 인증 | P0 |
| 2 | KYC 본인인증 | 신분증 업로드 → 관리자 승인 | P0 |
| 3 | JWT 토큰 인증 | Access Token + Refresh Token | P0 |
| 4 | 세션 관리 | 중복 로그인 방지, 자동 로그아웃 | P1 |
| 5 | 비밀번호 보안 | bcrypt 해싱, 최소 8자 정책 | P0 |

**KYC 인증 플로우:**
```
회원가입 → 신분증 업로드 → 관리자 검토 → 승인/거절
    ↓           ↓              ↓            ↓
  NONE      PENDING        REVIEWING    VERIFIED/REJECTED
```

| 7 | 역할별 접근 제어 | RolesGuard를 통한 API 보호 | P0 |
| 8 | Partner 할인 | 대량 구매 시 특별 할인율 적용 | P1 |

- **Invisible Tech**: 복잡한 기술은 숨기고 결과(PIN 발송)만 전달
- **Toss-like Clarity**: 텍스트 최소화, 큰 폰트와 직관적 아이콘

## 3. 비기능 요구사항

### 3.1 성능
- API 응답: < 200ms (95th percentile)
- 페이지 로드: < 3초 (LCP)
- 동시 사용자: 100명 이상 지원

### 3.2 보안
- HTTPS 필수
- PIN 코드 AES-256 암호화
- SQL Injection 방지 (Prisma ORM)
- XSS 방지 (React 자동 이스케이프)
- CSRF 토큰 (세션 기반)

### 3.3 가용성
- 가동 시간: 99.9% SLA
- PM2 클러스터 모드
- 자동 재시작 정책

## 4. 용어 정의

| 용어 | 설명 |
|------|------|
| **KYC** | Know Your Customer, 본인인증 |
| **PIN** | Personal Identification Number, 상품권 코드 |
| **Trade-In** | 매입, 고객이 보유한 상품권을 사이트에 판매 |

| **Partner** | 대량 구매 시 특별 할인을 받는 사업자 |
| **PG** | Payment Gateway, 결제 대행사 |
