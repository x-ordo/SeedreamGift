# WebAuthn + OTP 인증 체계 재구조화 설계

> **날짜**: 2026-03-27
> **상태**: 승인 대기
> **범위**: Go 서버 + 클라이언트 + 어드민 — 인증 흐름 전면 재설계

## 1. 목적

W기프트의 인증 체계를 **패스키(WebAuthn) 우선, 비밀번호+OTP 폴백** 구조로 재설계한다. 패스키는 로그인의 기본 수단으로, OTP는 비밀번호 로그인의 선택적 2차 인증으로, 비밀번호는 최후 복구 수단으로 포지셔닝한다.

### 인증 계층 구조

```
로그인 방법 1 (권장):  이메일 + 패스키 터치        → 2팩터 내장 (소유+생체)
로그인 방법 2 (폴백):  이메일 + 비밀번호 (+OTP)    → 1팩터 (+선택적 2팩터)
복구 경로:            비밀번호 재설정 (이메일 링크) → 패스키 재등록
최후 수단:            관리자 수동 초기화
```

### OTP의 역할 재정의

| 이전 | 이후 |
|------|------|
| 로그인 2차 인증 (MfaEnabled 시) | 비밀번호 로그인의 **선택적** 2차 인증 |
| 주문 시 OTP (이전 세션에서 구현) | **제거** — 패스키 세션이면 이미 2팩터, 비밀번호 세션이면 OTP는 로그인에서 처리 |
| 마이페이지에서 자유 on/off | 유지 — 사용자가 다음 로그인부터 OTP 사용 여부 선택 |

## 2. 로그인 흐름 상세

### 2.1 패스키 로그인 (권장)

```
사용자 → 이메일 입력 → "패스키로 로그인" 클릭
  → 서버: BeginAuthentication(email) → 챌린지 생성
  → 브라우저: WebAuthn ceremony (지문/Face ID/보안키)
  → 서버: FinishAuthentication(assertion) → 세션 발급
  → 완료: AccessToken + RefreshToken

보안 수준: 2팩터 (기기 소유 + 생체)
OTP 필요: ❌
```

### 2.2 비밀번호 로그인 (폴백/패스키 미등록)

```
사용자 → 이메일 + 비밀번호 입력 → "로그인" 클릭
  → 서버: Login(email, password)
  → IF user.MfaEnabled:
      → MFARequiredResponse 반환 (mfa_methods: ["totp"])
      → 프론트: OTP 6자리 입력 UI
      → 서버: LoginMFA(mfa_token, otp_code) → 세션 발급
  → ELSE:
      → 세션 즉시 발급

보안 수준: 1팩터 (비밀번호) 또는 2팩터 (비밀번호+OTP)
```

### 2.3 패스키 분실 → 비밀번호 복구 로그인

```
사용자 → "패스키를 사용할 수 없나요?" 클릭
  → 비밀번호 로그인 폼으로 전환
  → 비밀번호 입력 (+OTP if 활성화)
  → 로그인 성공 → 안내 배너: "새 패스키를 등록하세요"
  → 마이페이지에서 패스키 재등록
```

### 2.4 비밀번호 분실

```
사용자 → "비밀번호를 잊으셨나요?" 클릭
  → 이메일 입력 → 재설정 링크 발송
  → 새 비밀번호 설정 → 로그인 → 패스키 등록 유도
```

### 2.5 둘 다 분실 → 관리자 초기화

```
사용자 → 고객센터 문의
  → 관리자: 어드민 패널에서 사용자 검색
  → "WebAuthn 초기화" + "비밀번호 강제 재설정" 실행
  → 사용자에게 비밀번호 재설정 이메일 발송
```

### 5.2 패스키 관리 상세

**등록 (Create):**
- "새 패스키 등록" → 이름 입력 모달 ("예: 내 MacBook, 회사 PC") → WebAuthn ceremony → 성공 토스트
- `excludeCredentials` 설정으로 같은 기기 중복 등록 방지
- 등록 후 안내: "이 기기에서 패스키로 로그인할 수 있습니다"

**이름 변경 (Update):**
- 패스키 옆 연필 아이콘 클릭 → 인라인 편집 또는 모달 → PATCH 요청
- 유효성: 빈 문자열 불가, 50자 제한

**삭제 (Delete):**
- 삭제 버튼 클릭 → 확인 모달: "이 패스키를 삭제하면 해당 기기에서 패스키 로그인을 사용할 수 없습니다."
- 마지막 패스키 삭제 시 추가 경고: "모든 패스키가 삭제됩니다. 이후 비밀번호로만 로그인 가능합니다."
- 삭제 후 `WebAuthnEnabled` 자동 false (마지막 패스키인 경우)

**조회 (Read):**
- 이름, 등록일, 최근 사용일 표시
- 최근 사용일이 없으면 "아직 사용하지 않음"

### 5.3 OTP 관리 상세

**활성화:**
- OTP 토글 ON → QR 코드 + 시크릿 키 표시 → 6자리 코드 입력 → 활성화 완료
- 안내: "다음 비밀번호 로그인부터 OTP 인증이 요구됩니다. 패스키 로그인에는 영향이 없습니다."

**비활성화:**
- OTP 토글 OFF → 현재 OTP 코드 입력 → 비활성화 완료
- 안내: "OTP가 비활성화되었습니다. 비밀번호 로그인 시 추가 인증 없이 진행됩니다."

## 7. 백엔드 API 변경

### 7.1 로그인 흐름 변경

**`auth_service.go` Login() 메서드:**

```go
// 변경 전: MfaEnabled || hasWebAuthn → MFA 토큰
// 변경 후: MfaEnabled만 → MFA 토큰 (OTP 전용)
// WebAuthn은 별도 패스키 로그인 흐름으로 분리

if user.MfaEnabled {
    methods := []string{"totp"}
    // MFA 토큰 발급 + MFARequiredResponse 반환
}
// MfaEnabled가 아니면 바로 세션 발급
```

**패스키 로그인 라우트 유지 (이전에 삭제했던 것 복원):**

```go
waPublic.POST("/login/begin", h.WebAuthn.BeginAuthentication)   // 복원
waPublic.POST("/login/complete", h.WebAuthn.FinishAuthentication) // 복원
```

## 10. 보안 고려사항

| 위협 | 대응 |
|------|------|
| 패스키 로그인 브루트포스 | WebAuthn은 챌린지 기반이므로 브루트포스 불가. 비존재 이메일에도 동일 에러 반환 |
| 비밀번호 로그인 브루트포스 | 기존 IP 기반 10회 차단 (15분) 유지 |

| 패스키 분실 후 계정 탈취 | 비밀번호 로그인으로 복구 → OTP 활성화 시 2팩터 유지 |
| 관리자 초기화 남용 | 감사 로그에 관리자 ID + 사유 기록 |
| WebAuthn Replay | SignCount 검증으로 복제 키 탐지 |
| 크로스 사이트 패스키 사용 | rpID(Relying Party ID) 도메인 바인딩 |

  → [✓ 저장] [✗ 취소] 버튼

## 18. 검증

### 클라이언트
1. **패스키 로그인**: 이메일 → 패스키 버튼 → ceremony → 세션 → 대시보드
2. **패스키 실패 → 폴백**: ceremony 취소 → "비밀번호로 로그인" → 성공
3. **비밀번호 + OTP**: MFA 활성화 → 비밀번호 → OTP 6자리 → 성공
4. **비밀번호 단독**: OTP 비활성화 → 비밀번호만으로 성공
5. **회원가입 패스키**: 가입 → Step 3 → 패스키 등록 → 패스키로 로그인
6. **패스키 이름변경**: 마이페이지 → 편집 → 저장 → 반영
7. **주문**: OTP 모달 없이 바로 결제 진행
8. **브라우저 미지원**: 패스키 버튼 숨김

### 어드민
9. **패스키 로그인**: 어드민 로그인 → 패스키 → 관리자 대시보드
10. **비밀번호 + OTP**: 패스키 없이 → 비밀번호 + OTP → 성공
11. **사용자 WebAuthn 초기화**: 사용자 상세 → 초기화 → 해당 사용자 패스키 전삭
12. **사용자 OTP 강제 해제**: 사용자 상세 → 비활성화 → MFA 해제
13. **패스키 이름변경**: 보안설정 → 편집 → 저장

### 파트너
14. **패스키 로그인**: 파트너 로그인 → 패스키 → 파트너 대시보드
15. **비밀번호 + OTP**: 패스키 없이 → 비밀번호 + OTP → 성공
16. **OTP 설정**: 프로필 → OTP 토글 ON → QR 스캔 → 6자리 → 활성화
17. **OTP 해제**: 프로필 → OTP 토글 OFF → 코드 입력 → 비활성화
18. **패스키 이름변경**: 프로필 → 편집 → 저장

### 공통
19. **Go 빌드**: `cd go-server && go build ./...` 성공
20. **Client TS**: `cd client && npx tsc --noEmit --project tsconfig.app.json` 에러 없음
21. **Admin TS**: `cd admin && npx tsc --noEmit` 에러 없음
22. **Partner TS**: `cd partner && npx tsc --noEmit` 에러 없음
